渗透测试的大概流程和思路

渗透测试（Penetration Testing）是一种模拟攻击者的安全测试技术。它旨在帮助组织发现和修复存在的安全漏洞，并加强网络的安全性。

一般来说，渗透测试的流程如下：

  1. 计划和准备：包括识别测试目标、了解测试目标的基本信息、制定测试计划和测试方案。确定测试的目标和范围，并准备好相关的测试工具。
  2. 收集信息：通过各种手段收集目标系统的信息，包括网络拓扑、操作系统版本、常用软件、开放端口和服务等。
  3. 扫描和枚举：利用自动化工具对目标系统进行扫描和枚举，以确定系统的漏洞和弱点。
  4. 攻击尝试：利用扫描枚举出的漏洞和弱点进行攻击，以确定系统的安全性。
  5. 权限提升：在获取了目标系统的控制权限后，尝试提升权限，以便获取更多系统信息和功能。
  6. 后渗透：在获得更高的权限后，进一步渗透目标系统，手机更多信息，并寻找新的漏洞进行攻击。
  7. 报告编写：根据测试结果编写报告，并提供建议和修复方案。

在进行渗透测试时，需要遵循一些基本原则，包括：

  • 合法合规：渗透测试应符合法律法规，并征得测试目标的同意。
  • 保护数据安全：在进行测试时，应保护测试目标的数据安全，避免造成不必要的损失。
  • 不影响业务的正常使用

其他测试流程思路

明确目标→信息收集→漏洞探测→漏洞验证→信息分析→获取所需→信息整理→形成报告

1、明确目标：
  • 确认范围：测试目标的范围，IP，域名，内外网。
  • 确认规则：能渗透到什么程度，时间，能否提权等。
  • 确认需求：WEB应用的漏洞（新上线程序）？业务逻辑漏洞（针对业务的）？人员权限管理漏洞（针对人员、权限）？等等，要做到立体全方位。
2、信息收集：
  • 方式：主动扫描，开放搜索：利用搜索引擎获得后台，未授权页面，敏感url等。
  • 基础信息：真实IP，网段，域名，端口。
  • 系统信息：操作系统的版本。
  • 应用信息：各端口的应用，例如WEB应用，邮件应用等。
  • 版本信息：所有这些探测到的东西的版本。
  • 人员信息：域名注册人员信息，WEB应用中网站发帖人的id，管理员姓名等。
  • 防护信息：试着看能否探测到防护设备。
3、漏洞探测：
  • 方法：使用漏扫，例如：AWVS、IBM appscan等。
  • 结合漏洞去exploit-db等网址找利用。
  • 在网上寻找验证poc。
  • 内容：系统漏洞：系统没有及时打补丁。
  • Websever漏洞：Websever配置问题。
  • Web应用漏洞：Web应用开发问题。
  • 其它端口服务漏洞
  • 通信安全
4、漏洞验证：
  • 自动化验证：结合自动化扫描工具提供的结果。
  • 手动验证：根据公共资源进行验证。
  • 试验验证：自己搭建模拟环境进行验证。
  • 登陆猜解：有时可以尝试猜解一下登陆的账号密码等信息。
  • 业务漏洞验证：如发现业务漏洞，要进行验证。
5、信息分析：
  • 精准打击：准备好上一步探测到的漏洞的exp，用来精准打击。
  • 绕过防御机制：是否有防火墙等设备，如何绕过。
  • 定制攻击路径：最佳工具路径，根据薄弱入口，高内网权限位置，最终目标。
  • 绕过检测机制：是否有检测机制，流量监控，杀毒软件，恶意代码检测等（免杀）。
  • 攻击代码：经过试验得来的代码，包括不限于xss代码，sql注入语句等。
6、获取所需：
  • 实施攻击：根据前几步的结果，进行攻击。
  • 获取内部信息：基础设施（网络连接，路由，拓扑等）。
  • 进一步渗透：内网入侵，敏感目标。
  • 持续性存在：一般我们对客户做渗透不需要。rootkit，后门，添加管理账号等。
  • 清处痕迹：清理相关日志（访问，操作），上传文件等。
7、信息整理：
  • 整理渗透工具：整理渗透过程中用到的代码，poc，exp等。
  • 整理收集信息：整理渗透过程中收集的一切信息。
  • 整理漏洞信息：整理渗透过程中遇到的各种漏洞，各种脆弱位置信息。
  • 目的：为了最后形成报告，形成测试结果使用。
8、形成报告：
  • 按需整理：按照之前第一步跟客户确定好的范围，需要来整理资料，并将资料形成报告。
  • 补充介绍：要对漏洞成因，验证过程和带来危害进行分析。
  • 修补建议：当然要对所有产生的问题提出合理高效安全的解决办法。
  • 渗透测试报告编写：
  • 封皮
  • 明确重点和要求
  • 给谁看？不同职位的人关注的侧重点不一样
  • 内容提要
  • 漏洞列表（一页纸内）
  • 过程清晰准确
  • 工具（版本和功能）
  • 渗透测试团队成员及联系方式
  • 修补建议

像百度等大型网站一般都有cdn，怎样寻找它的真实ip

  • 通过网站的whois查询，查询域名所对应的IP地址。
  • 使用nslookup工具，查询域名对应的DNS记录。
  • 通过网络扫描工具，扫描网站对应的IP地址段，寻找网站真实IP。
  • 通过网站的源代码查看，查看是否有指向真实IP的链接。
  • 利用抓包工具抓取网站的网络流量，查看是否有指向真实IP的信息。

其他绕过cdn查找真实ip的方法

  • 在ip138上查询历史DNS解析记录：https://site.ip138.com/

  • 在﻿站长之家上ping测试：http://ping.chinaz.com/

  • 使用国外主机名解析域名：国内很多 CDN 厂商因为各种原因只做了国内的线路，而针对国外的线路可能几乎没有，此时我们使用国外的主机直接访问可能就能获取到真实IP。

  • 查询子域名：毕竟 CDN 还是不便宜的，所以很多站长可能只会对主站或者流量大的子站点做了 CDN，而很多小站子站点又跟主站在同一台服务器或者同一个C段内，此时就可以通过查询子域名对应的 IP 来辅助查找网站的真实IP。

  • 网站邮件订阅查找：RSS邮件订阅，很多网站都自带 sendmail，会发邮件给我们，此时查看邮件源码里面就会包含服务器的真实 IP 了。或者通过网站注册等功能接收网站确认邮件。2.6、手机APP端抓包：手机端的数据包抓到的可能是服务器真实IP。

搜集网站的指纹信息，你一般会怎么利用呢

指纹搜集

网站指纹是 Web 服务组件在开发时留下的类型、版本等标记信息，包括 Web 服务器指纹、Web 运用指纹及前端框架指纹等。

除了前面用 Nmap 扫描端口和服务的方法，我们还可以通过前端网页和返回一些 HTTP 头信息来判断网站使用的哪些开发框架、服务器、系统，在渗透测试时也可以提供有针对性的测试思路。

推荐 Chrome 插件Wappalyzer。在你打开网站后，点击插件，按钮就会弹出如下类似信息，网站的指纹信息一目了然。

一般搜集网站的指纹信息后，我们可以利用这些信息进行如下操作：

  1. 通过指纹信息进行网站版本识别，确定网站使用的是哪个版本的软件，进而判断网站是否存在已知漏洞。
  2. 通过指纹信息进行网站指纹识别，确定网站的架构和服务器类型，进而制定攻击策略。
  3. 通过指纹信息进行网站功能识别，确定网站提供的功能和服务，进而进行更细致的安全检测。
  4. 通过指纹信息进行模糊测试，寻找网站未曝光的漏洞，并进行利用。
  5. 通过指纹信息进行攻击模拟，模拟网站遭受攻击的情况，并验证网站的安全防护能力。

讲一下sql注入的原理吧

SQL注入攻击是一种恶意的攻击，目的是通过插入恶意的SQL代码来破坏数据库的完整性，或者获取数据库中的敏感信息。

通常，这种攻击是通过提交表单或者URL参数的形式来实现的，攻击者会在提交的数据中包含恶意的SQL代码，被数据库解释器执行。由于数据库解释器无法区分正常的查询语句和恶意代码，因此会将恶意代码作为查询语句来执行，从而达到破坏数据库或获取敏感信息的目的。

为了防止这种攻击，应该遵循一些基本的安全措施，例如对用户提交的数据进行严格的过滤和验证，确保其中不包含恶意的SQL代码，并使用参数化查询来代替拼接SQL语句的方式，这样可以有效地防止SQL注入攻击。

有了解过宽字节注入吗？

宽字节注入是恶意攻击手段，它利用了程序对宽字节字符串的不当处理，来注入恶意代码。通过插入特殊字符或者字符串，使程序识别出一些额外信息，导致程序发生异常或漏洞，从而获取控制权限或者访问敏感信息。宽字节注入的主要原理是利用程序对于特殊字符的不完全处理，从而达到攻击目的。

宽字节字符串是指使用多字节字符集编码的字符串，其中每个字符占用多个字节。这种类型的攻击通常发生在使用宽字节字符集的语言（如中文、日文等）中，因为这些语言使用的字符集中的字符通常需要多个字节来表示。

mysql的udf提权，你有了解过吗？

MySQL UDF 是一种特殊的 MySQL 函数，它允许用户在自己的代码中定义函数。这对于提高 MySQL 服务器的性能和扩展其功能非常有用。

由于UDF有提权漏洞，如果恶意用户利用该漏洞，可能导致提权。

提权的方法为：

  1. 首先，恶意用户需要创建一个可以执行系统命令的UDF函数，可以使用C语言来实现。
  2. 然后，恶意用户需要在MySQL中创建一个新的函数，并将其指向刚才创建的UDF函数。
  3. 最后，恶意用户可以通过执行该函数来获取提权，并执行系统命令。

为了防止UDF提权漏洞的发生，MySQL提供了一个叫做“secure_file_priv”的系统变量，可以限制UDF函数的存储位置，或者在MySQL中禁用UDF函数,从而避免恶意用户利用UDF提权的情况。

如果你想利用udf提权，需要确认mysql版本吗？追问，哪个版本之上和之下有什么区别你知道吗？

是的，如果你想利用 UDF 提权，你需要确认 MySQL 的版本。不同的版本有不同的特性，并且一些功能可能只在特定的版本中可用。因此，确认 MySQL 版本的重要性。

更重要的是

MySQL 5.0版本之上开始支持udf，而MySQL 5.7版本之下不支持udf，因此如果目标服务器的mysql版本低于5.7，那么可以考虑使用udf提权。

在MySQL 5.7版本之上，udf功能被禁用，需要通过特殊方式才能启用udf，这就需要更高的技巧和知识。

总的来说，udf提权的可行性取决于mysql版本，需要对目标服务器的版本进行确认。

MYSQL 5.1以上版本，必须要把udf.dll文件放到MYSQL安装目录下的libplugin文件夹下才能创建自定义函数

因为不同版本的mysql对udf的支持情况不同，在不同版本中利用udf提权的方式也不同。

通常来说，在mysql 5.1版本之上，udf提权的方式一般是利用mysql的函数提权漏洞，通过在udf中注入代码实现提权。而在mysql 5.1版本之下，udf提权的方式则一般是通过利用mysql的反序列化漏洞实现提权。
